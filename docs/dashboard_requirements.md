# Dashboard Requirements

## Project Name

Modern Financial Analytics Pipeline

---

## Dashboard Purpose

The purpose of this dashboard is to provide a clear analytical view of a simulated financial loan portfolio.

The dashboard is designed to help business users monitor loan performance, payment behavior, portfolio risk, advisor performance, branch performance, and customer segment trends.

This dashboard will be built using the analytics outputs generated by the local SQLite analytics layer.

---

## Business Questions

The dashboard should help answer the following business questions:

1. What is the total loan portfolio size?
2. How many loans are active, closed, or delinquent?
3. What is the total disbursed loan amount?
4. What is the average loan amount?
5. Which branches manage the largest loan portfolios?
6. Which advisors manage the largest loan portfolios?
7. Which products represent the highest loan exposure?
8. Which customer segments represent the largest portion of the portfolio?
9. What is the monthly payment trend?
10. What is the distribution of paid, late, and missed payments?
11. Which loans show higher payment risk?
12. Which branches or advisors show higher delinquency behavior?

## Recommended Power BI Measures

The following DAX measures are recommended for the Power BI model.

```DAX
Total Loans = COUNTROWS(loan_level_analytics)

Total Loan Amount = SUM(loan_level_analytics[loan_amount])

Average Loan Amount = AVERAGE(loan_level_analytics[loan_amount])

Total Paid Amount = SUM(loan_level_analytics[total_paid_amount])

Average Days Late = AVERAGE(loan_level_analytics[average_days_late])

Late Payment Count = SUM(loan_level_analytics[late_payment_count])

Missed Payment Count = SUM(loan_level_analytics[missed_payment_count])

Delinquent Loans =
CALCULATE(
    COUNTROWS(loan_level_analytics),
    loan_level_analytics[loan_status] = "Delinquent"
)

Delinquency Rate =
DIVIDE(
    [Delinquent Loans],
    [Total Loans]
)

Payment Recovery Ratio =
DIVIDE(
    [Total Paid Amount],
    [Total Loan Amount]
)
```

## Design Recommendations

The dashboard should follow these design principles:

1. Use a clean executive layout.
2. Place KPIs at the top of each page.
3. Use consistent number formatting.
4. Use currency formatting for monetary values.
5. Use percentage formatting for rates.
6. Avoid overcrowding pages with too many visuals.
7. Use filters and slicers consistently.
8. Use conditional formatting to highlight risk.
9. Prioritize business readability over visual complexity.
10. Keep page navigation simple.

---

## Suggested Color Logic

Color should support interpretation, not decoration.

Suggested semantic logic:

| Meaning | Suggested Usage |
|---|---|
| Positive / healthy | Paid loans, recovered amounts, closed loans |
| Warning | Late payments, moderate risk |
| Critical | Missed payments, delinquent loans |
| Neutral | Totals, inactive comparisons, general categories |

Specific colors can be selected later during the Power BI design phase.

## Dashboard Success Criteria

The dashboard will be considered successful if it allows users to:

1. Understand the total portfolio size quickly.
2. Identify loan exposure by branch, advisor, product, and customer segment.
3. Monitor payment behavior over time.
4. Detect late and missed payment patterns.
5. Review delinquent loan behavior.
6. Filter portfolio performance by key business dimensions.
7. Support executive-level and operational-level analysis.
8. Connect clearly to the underlying pipeline outputs.
