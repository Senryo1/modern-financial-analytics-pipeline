# Power BI Build Guide

## Project Name

Modern Financial Analytics Pipeline

---

## Purpose

This document provides a practical guide for building the Power BI dashboard based on the analytics outputs generated by the project pipeline.

The dashboard will be built using CSV files exported by:

```text
src/run_sqlite_analytics.py
```

Source folder:

```text
data/analytics/
```

---

## Recommended Build Strategy

The first version of the Power BI dashboard should use the generated analytics CSV outputs instead of connecting directly to the SQLite database.

This approach is recommended because:

1. The analytics logic has already been prepared by the pipeline.
2. The CSV files are easier to load and inspect.
3. The dashboard can focus on business analysis instead of data preparation.
4. The project clearly separates transformation, validation, database loading, analytics logic, and reporting.
5. Future versions can connect directly to SQLite, SQL Server, PostgreSQL, Snowflake, or dbt models.

## Recommended Model Strategy

For the first dashboard version, avoid creating unnecessary relationships between all summary tables.

Use:

```text

```

as the main detailed analytical table.

Use the other tables as pre-aggregated outputs for specific visuals.

This avoids model complexity and keeps the first version simple, stable, and easy to explain.

# Recommended DAX Measures

Create the following measures using the `Loan Level Analytics` table.

## Core Portfolio Measures

```DAX
Total Loans =
COUNTROWS('Loan Level Analytics')
```

```DAX
Total Loan Amount =
SUM('Loan Level Analytics'[loan_amount])
```

```DAX
Average Loan Amount =
AVERAGE('Loan Level Analytics'[loan_amount])
```

```DAX
Total Paid Amount =
SUM('Loan Level Analytics'[total_paid_amount])
```

```DAX
Average Days Late =
AVERAGE('Loan Level Analytics'[average_days_late])
```

---

## Risk and Payment Measures

```DAX
Late Payment Count =
SUM('Loan Level Analytics'[late_payment_count])
```

```DAX
Missed Payment Count =
SUM('Loan Level Analytics'[missed_payment_count])
```

```DAX
Delinquent Loans =
CALCULATE(
    COUNTROWS('Loan Level Analytics'),
    'Loan Level Analytics'[loan_status] = "Delinquent"
)
```

```DAX
Delinquency Rate =
DIVIDE(
    [Delinquent Loans],
    [Total Loans]
)
```

```DAX
Payment Recovery Ratio =
DIVIDE(
    [Total Paid Amount],
    [Total Loan Amount]
)
```

## Dashboard Design Notes

Use a clean and professional style.

Recommended design principles:

1. Use a simple executive layout.
2. Avoid excessive visuals on one page.
3. Use consistent titles.
4. Align all KPI cards.
5. Use readable labels.
6. Format currency and percentages correctly.
7. Use conditional formatting for risk indicators.
8. Keep colors semantic and consistent.
9. Avoid unnecessary decorative elements.
10. Prioritize business clarity.
